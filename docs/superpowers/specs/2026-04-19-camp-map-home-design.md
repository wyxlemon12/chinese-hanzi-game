# 营地探险首页地图改版设计

## 背景

当前首页已经具备项目冒险、自由单字和新探险地图三个入口，但主视觉仍然偏卡片流，缺少“探险地图”应有的空间感和任务感。用户希望把首页升级成一个更漂亮、更像游戏地图的入口，并明确要求：

- 首页顶部使用 SVG 地图作为主视觉
- 地图上不要直接显示单字
- 地图上的点击对象改成 10 张散落的小线索卡
- 点击线索卡后直接进入写字动画页
- 到写字动画页后才第一次揭晓单字
- 首页底部仍保留 `自由单字` 和 `新探险地图` 两个次要入口

本次设计只覆盖首页地图入口和地图到学习页的衔接，不重写整个学习主流程。

## 目标

- 把首页升级成“地图主入口”，让项目冒险不再只是普通卡片
- 在不剧透单字的前提下，让用户知道每个线索点代表什么主题任务
- 复用现有学习页能力，减少改动范围
- 保持现有 `自由单字` 和 `新探险地图` 的可用性

## 非目标

- 不引入后端、Supabase 或外部生成服务
- 不在本轮重做 `GeneratedMapScreen` 为完整 SVG 地图
- 不在首页展示古诗卡内容
- 不把单字提前暴露在首页地图上
- 不把首页改成全屏沉浸地图页；仍保留次要入口区

## 用户体验摘要

首页进入后，用户首先看到一张营地探险地图。地图背景带有地形块、路径和营地环境元素，但中央不再放主营地节点。地图上分布 10 张散落的小线索卡，每张卡表现为：

- 线索编号
- 具象主题图标
- 短任务名

这些卡片是地图上的主要点击对象。点击后：

- 不弹中间确认层
- 直接进入写字动画页
- 在写字动画页顶部先显示“当前线索任务”
- 然后揭晓这次学习的单字
- 再进入现有的动画、描写、完成、古诗卡流程

## 信息架构

首页结构调整为三层：

1. 地图主视觉区
2. 次要入口区
3. 底部导航

### 1. 地图主视觉区

地图主视觉区占据首页顶部主要空间，承担项目冒险的主入口职责。其内部包含：

- 页面标题与一句简短说明
- 营地地图背景
- 10 张散落的线索卡

### 2. 次要入口区

地图下方保留两个较轻量的入口卡片：

- `自由单字`
- `新探险地图`

这两个入口不抢地图主视觉，但仍然清晰可达。

### 3. 底部导航

保持现有底部导航结构不变，减少整体框架改动。

## 首页地图视觉设计

### 总体风格

首页地图采用“营地探险”语义，但不做过度拟真。视觉上保持：

- 柔和的大地色底图
- 轻微起伏的地形块
- 几条路径串联不同区域
- 少量树丛、坡地、水域等抽象形状

目标是“像地图”，而不是“像海报”或“像列表背景”。

### 地图背景

地图背景是一个固定尺寸的 SVG 容器，内部由下列图层组成：

- 底层地形底色
- 若干地貌色块
- 路径 / 小路
- 轻量装饰元素
- 线索卡层

背景图层只提供环境和方向感，不承载文字信息。

## 线索卡设计

### 卡片类型

首页使用 `A1` 方案：地图背景上散落的 10 张可点击小卡片。

### 卡片内容

每张卡片固定包含：

- `线索 01` 这类编号
- 一个具象主题图标
- 一条非常短的任务名

卡片不显示：

- 单字本体
- 拼音
- 古诗信息

### 图标策略

地图点位采用用户最终确认的 `A · 具象主题图标` 路线，不使用徽章外圈版本作为首版默认。

主题图标根据任务语义映射，例如：

- 树 / 林地
- 山 / 山路
- 太阳 / 清晨
- 水滴 / 溪流
- 云 / 天空
- 风 / 风痕
- 足迹 / 路线
- 人影 / 伙伴

### 排布方式

10 张卡片不是规则网格，而是在地图上错落散布，要求：

- 保持可点击热区足够大
- 避免互相遮挡
- 维持清晰阅读顺序
- 看起来像“地图上的任务点”，不是普通运营卡片流

### 状态设计

每张线索卡存在三种状态：

- 未完成
  - 更明亮
  - 对比更高
  - 可直接点击
- 进行中
  - 有更明显的描边或高亮
  - 表示当前推荐继续的线索
- 已完成
  - 略微压暗或降低强调度
  - 保留点击能力

首版不要求复杂动画，只要状态区分清晰即可。

## 地图点击交互

地图点位的点击逻辑固定为：

- 点击线索卡
- 直接进入写字动画页
- 不出现中间预览卡
- 不在首页预先揭晓单字

这样首页负责“接任务”，学习页负责“揭晓答案”。

## 写字动画页承接

写字动画页沿用现有 `LessonExperience` 主流程，只新增一层线索语境承接。

### 进入页顶部新增内容

写字动画页顶部新增：

- 线索编号
- 任务短名
- 简短任务说明

示例：

- `线索 04`
- `任务：找到通往山顶的山路标记`

### 单字揭晓时机

用户最终确认采用“直接揭晓”：

- 点击地图卡后直接进入写字动画页
- 到达该页后立即揭晓单字
- 然后进入现有动画与描写流程

### 其余学习流程

保留现有：

- 动画演示
- 描写
- 完成反馈
- 古诗卡

只是在顶部加入任务语境，不重写学习引擎。

## 数据设计

本次设计不引入后端，仅补充前端线索数据字段。

每个项目关卡需要提供或派生以下字段：

- `clueIndex`
- `clueTitle`
- `clueIcon`
- `clueDescription`

其中：

- `clueStatus` 可由现有进度快照派生
- `clueIcon` 通过本地映射表渲染

### 示例结构

```ts
type ClueMeta = {
  clueIndex: number;
  clueTitle: string;
  clueDescription: string;
  clueIcon: 'tree' | 'mountain' | 'sun' | 'water' | 'cloud' | 'wind' | 'trail' | 'person';
};
```

这些数据可以挂在现有项目课程配置上，而不必引入新的独立存储层。

## 组件与实现边界

### 1. `HomeScreen`

`HomeScreen` 是本次改动的主战场。

需要新增：

- 一个地图主视觉区组件
- 线索卡渲染
- 点击线索卡进入项目学习

保留：

- 自由单字入口
- 新探险地图入口

### 2. 新增地图组件

建议新增单独组件，例如：

- `CampAdventureMap.tsx`
- `ClueMapCard.tsx`

组件边界：

- `CampAdventureMap`
  - 负责背景 SVG
  - 负责 10 张卡片布局
  - 负责将点击事件抛给上层
- `ClueMapCard`
  - 负责单张卡片样式和状态

### 3. `LessonExperience`

只做小幅增强：

- 新增顶部线索语境区
- 接收线索元数据

不改动核心写字流程。

## 数据流

### 首页

首页从当前项目课程中拿到：

- 当前 10 个线索项
- 每个线索的状态

然后渲染成地图卡片。

### 点击后

点击地图卡片时：

- 使用现有项目学习入口逻辑打开对应 lesson
- 同时把 clue metadata 传入 `LessonExperience`

### 学习页

`LessonExperience` 显示 clue metadata，再显示单字和现有学习内容。

## 错误与回退策略

- 如果某个线索项缺少 `clueIcon`，用默认图标回退
- 如果缺少 `clueTitle`，用 `线索 n` 回退
- 如果学习页未收到 clue metadata，仍然允许按当前 lesson 正常学习
- 地图组件布局失败时，仍要保证线索卡能渲染为简单列表，而不是整页空白

## 可访问性与移动端要求

- 卡片点击区域必须适合移动端手指点击
- 图标不能是唯一信息来源，编号和任务名仍需保留
- 卡片状态要靠颜色 + 层级双重表达，而不是只靠颜色差别
- 首页地图在窄屏下优先保证点击性，而不是追求过多装饰

## 测试计划

### 组件测试

- 首页地图能渲染 10 张线索卡
- 点击线索卡会调用现有项目 lesson 打开逻辑
- 已完成 / 未完成 / 进行中状态样式可区分

### 页面测试

- 首页仍保留 `自由单字` 和 `新探险地图`
- 首页地图不显示单字文本
- 学习页顶部会显示线索任务信息

### E2E

- 从首页点击一个线索卡
- 成功进入写字动画页
- 到学习页才看到对应单字
- 完成学习后主流程不回归

### 回归

- 项目冒险原流程不崩
- 自由单字不受影响
- 新探险地图不受影响
- `npm run test:ci`
- `npm run test:e2e`
- `npm run lint`
- `npm run build`

## 分阶段实施建议

### 第 1 阶段

- 完成首页地图主视觉
- 完成 10 张线索卡点击进入学习页

### 第 2 阶段

- 为学习页接入线索语境区
- 优化状态样式和当前线索高亮

### 第 3 阶段

- 如效果满意，再考虑把 `GeneratedMapScreen` 也统一成地图风格

## 决策记录

本次已确认的关键决策如下：

- 风格：营地探险地图
- 首页入口：地图作为顶部主视觉
- 地图组织：不使用中心营地节点作为主结构
- 地图点位：改为 10 张散落小卡片
- 图标策略：采用 `A · 具象主题图标`
- 首页不显示单字
- 点击线索卡后直接进入写字动画页
- 单字在学习页中揭晓
- 首页保留 `自由单字` 与 `新探险地图` 次要入口
